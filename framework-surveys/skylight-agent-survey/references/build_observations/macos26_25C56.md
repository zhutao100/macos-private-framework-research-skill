# SkyLight Observations: macOS 26.2 25C56

Scope: local arm64 VM, macOS 26.2 build 25C56, Darwin 25.2.0.

## Host Facts

- Primary dyld cache: `/System/Volumes/Preboot/Cryptexes/OS/System/Library/dyld/dyld_shared_cache_arm64e`.
- Xcode: 26.4 build 17E192.
- `ipsw`: 3.1.682.
- SIP and authenticated root were enabled during collection.

## Framework Packaging

- `/System/Library/PrivateFrameworks/SkyLight.framework/Versions/A/` contains resources and `_CodeSignature`, but no `SkyLight` binary file.
- `/System/Library/PrivateFrameworks/SkyLight.framework/SkyLight` is therefore a symlink whose terminal binary is absent from the filesystem.
- The dyld cache map still contains `/System/Library/PrivateFrameworks/SkyLight.framework/Versions/A/SkyLight`.
- `dlopen("/System/Library/PrivateFrameworks/SkyLight.framework/SkyLight", RTLD_LAZY | RTLD_LOCAL)` succeeds on this build, resolving through the dyld shared cache.

Implication: tools that call `file`, `nm`, `dyldinfo`, or `strings` on the framework symlink can fail even when runtime loading works. Prefer dyld-cache-aware tooling for SkyLight on this build.

## Extraction And Headers

- Plain `ipsw dyld extract <cache> SkyLight -o <dir> --force` produced a flat extracted file named `SkyLight`.
- The extracted binary is arm64e and has UUID `E7D21A4B-D787-3703-9CF3-7A6AD756224B`.
- Install name: `/System/Library/PrivateFrameworks/SkyLight.framework/Versions/A/SkyLight`.
- Current version from load commands: `600.0.0`.
- `ipsw class-dump <cache> SkyLight --headers --output <dir>` emitted 301 Objective-C header files.

- `ipsw class-dump <extracted>/SkyLight --headers --output <dir>` failed on the flat extracted file with an Objective-C metadata address mapping error. Use the dyld cache path for class-dump on this build, and keep the extracted file for Mach-O load-command and symbol inspection.

## RuntimeViewer Cross-Check

RuntimeViewer 2.0.1 was available at `/Applications/RuntimeViewer.app`. A RuntimeViewerCore local-engine probe loaded `/System/Library/PrivateFrameworks/SkyLight.framework/SkyLight`, enumerated 332 runtime objects, and exported rich interfaces for the 10 class names used by the survey probe.

Compared with `ipsw class-dump`, RuntimeViewer added useful project-triage evidence for selected Objective-C classes:

- ivar offsets for classes such as `SLContentFilter` and `SLSEventAuthenticationMessage`;
- method and property-accessor IMP addresses;
- stronger property storage evidence for bridged Space/window operation classes.

Selected RuntimeViewer observations:

- `SLSBridgedCopySpacesForWindowsOperation` stores `unsigned int _options` and `NSArray *_windows`; its initializer selector is `initWithOptions:windows:`.
- `SLContentFilter` stores `NSSet`-typed included/excluded window/application/PID sets and exposes mutable `includedPIDS`, `excludedPIDS`, `hideMenuBar`, and `enforceSharingType` properties.
- `SLSEventAuthenticationMessage` stores event-type, target-process, proxy-target-process, signature, timestamp, origin, context, and capability fields; it remains event-authentication evidence only, not a callable surface in this package.

## Hopper Cross-Check

Hopper 6.2.9 analyzed the extracted arm64e SkyLight image. The focused snapshot reported 23,245 procedures total and completed background analysis.

Read-only declaration evidence:

- `SLSCopyManagedDisplays` and `SLSCopyManagedDisplaySpaces` each take one connection argument in the fallback path and return bridged result arrays (`strings` and `propertyListArray`) when bridged window-management operations are enabled.
- `SLSCopySpacesForWindows` takes option/window-array arguments and returns a bridged `numbers` result when bridged operations are enabled.
- `SLSCopyWindowsWithOptionsAndTags` preserves the six-argument shape used by `SkyLightReadOnly.h`; Hopper showed the bridged initializer `initWithOwner:spaces:options:setTags:clearedTags:`.
- `SLSSpaceGetType` and `SLSManagedDisplayGetCurrentSpace` preserve the two-argument shapes used by the header.
- `SLSGetWindowOwner` and `SLSGetWindowLevel` write through output pointers and return a status-like integer.
- `SLSGetWindowBounds` is a thin wrapper over an internal `_GetWindowBounds` call.
- `SLSGetActiveSpace` does not read an explicit connection argument in the macOS 26.2 build 25C56 Hopper pseudocode; Hammerspoon caller evidence and this package's runtime validation still pass a connection ID. Keep the header's connection-argument form for source compatibility and cross-build caution.

Focused event-authentication evidence:

- `+[SLSEventAuthenticationMessage messageWithEventRecord:pid:version:]` calls `classForEventType:` using the event record and then calls `initWithEventRecord:pid:version:`.
- `-[SLSEventAuthenticationMessage initWithEventRecord:pid:version:]` delegates through `initWithMessageInitData:`.
- This corroborates the class-dump selector shape but does not validate the full `SLSEventRecord` layout for project use.

# SkyLight Observations: macOS 26.2 25C56

Scope: local arm64 VM, macOS 26.2 build 25C56, Darwin 25.2.0.

## Host Facts

- Primary dyld cache: `/System/Volumes/Preboot/Cryptexes/OS/System/Library/dyld/dyld_shared_cache_arm64e`.
- Xcode: 26.4 build 17E192.
- `ipsw`: 3.1.682.
- SIP and authenticated root were enabled during collection.

## Framework Packaging

- `/System/Library/PrivateFrameworks/SkyLight.framework/Versions/A/` contains resources and `_CodeSignature`, but no `SkyLight` binary file.
- `/System/Library/PrivateFrameworks/SkyLight.framework/SkyLight` is therefore a symlink whose terminal binary is absent from the filesystem.
- The dyld cache map still contains `/System/Library/PrivateFrameworks/SkyLight.framework/Versions/A/SkyLight`.
- `dlopen("/System/Library/PrivateFrameworks/SkyLight.framework/SkyLight", RTLD_LAZY | RTLD_LOCAL)` succeeds on this build, resolving through the dyld shared cache.

Implication: tools that call `file`, `nm`, `dyldinfo`, or `strings` on the framework symlink can fail even when runtime loading works. Prefer dyld-cache-aware tooling for SkyLight on this build.

## Extraction And Headers

- Plain `ipsw dyld extract <cache> SkyLight -o <dir> --force` produced a flat extracted file named `SkyLight`.
- The extracted binary is arm64e and has UUID `E7D21A4B-D787-3703-9CF3-7A6AD756224B`.
- Install name: `/System/Library/PrivateFrameworks/SkyLight.framework/Versions/A/SkyLight`.
